Project Overview

This project required me to build a system to aid in capturing online feedback for primary research. The questionnaire system needed to have a secure administration backend for questionnaire and question creation as well as providing a view for administrators to see responses. The system was developed using Laravel. The questionnaire were made to be shown to users via an unsecured interface that can be developed in either Laravel or via AngularJS.

ERD Diagram

Developing an ERD diagram was the first system model created. The ERD diagram shows the structural integrity of the database and the relationships between each element.

Use Case Diagram

Use case diagrams were developed to model the system of the system. The use case identified the main tasks that can be completed and the steps to which that task would be carried out.

Class Diagram

A class diagram was developed to describe the structure of the system. This was done by identifying the systems classes, attributes, methods and the relationships between each table.

Sequence Diagram

Sequence diagrams were developed to show the process in which a task is completed, and if a sub-task cannot be authorised/completed then it shows the process then taken to correct it.

Frameworks

Composer

Although Composer is not a framework it was a very important section of the build. Composer is a dependency manager and therefore manages the PHP software and the libraries within it. As well as this it solves the autoloading inside of the PHP packages. For Composer to work on this application it needed to be installed at the very beginning of the build.

Laravel

The questionnaire application was built using Laravel due to it being able to create web applications and makes development tasks easier to complete due to the simple syntax. Laravel also provides a file structure of professional standards which when used for a large project like this one, was highly beneficial. It also makes it much easier to manage migrations, seeds, controllers, etcâ€¦ because of the pre-existing files and folders that are included. Authentication (login, register) is an important part of the questionnaire app and this is a feature that is implemented through Laravel.

Bootstrap 4

Bootstrap 4 is an open source CSS framework, and this was integrated inside of the Laravel framework to enable the CSS side of the build much more manageable. As well as providing CSS code it also enabled a fully responsive website quickly and easily without the developer (myself) having to ensure that this had been implemented.

Testing

To test the questionnaire application an automated approach was taken. Behaviour Driven Design (BDD) fucntional testing was undertaken to test the following objectives:

  • Ensure that the application, under testing, conforms to requirements
  • Ensure that the application meets some if not all of the specifications defined by the client

The tests completed follow what the user would see, click, where buttons navigate them to and what input fields they would fill in. The tests are as followed:

  • Create a Questionnaire and Questions
  • Login
  • Access questionnaire to complete it without needing to sign in
